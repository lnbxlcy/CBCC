    Distributed structure

    The block chain of the CBC fund builds a distributed structure system based on the open source, decentrating protocol established by the system to send the information of value exchange to the whole network through distributed communication, to determine the content of information data through distributed accounts, and to create block data after a timestamp, and then distribute the data through distributed transmission. Send to each node to implement distributed storage

    Data block structure

    In the CBC fund, data storage is also like block chain, which is stored permanently in blocks. Blocks are generated and linked one by one in chronological order, and each block records all transaction information during the creation period.。

    Consensus mechanism

    The CBC fund uses the proxy rights Byzantine fault tolerant algorithm (Delegated Stake-PBFT), referred to as DS-PBFT, which is a modified algorithm based on the PBFT consensus mechanism, which makes the weight of the verifier voting corresponding to the token interest on the chain on which it is mortgaged.

    Security encryption algorithm

    The CBC fund uses the homomorphic encryption technology to store the information of the transaction in the supply chain by the homomorphic encryption technology, and the information of its products can achieve the perfect balance, and will not cause any significant changes to the block chain properties.

    Centralization of autonomy

    With the accumulation of network data, the "CBC fund" needs to be upgraded constantly. It can be divorced from current institutional constraints or endorsement by third party organizations, and both sides can achieve value delivery directly. This "de centralization" feature can effectively reduce transaction costs, improve transaction efficiency and reduce frictions caused by transaction consistency.
